## Data Flow — End to End

### 1. Ingestion
A pollution event enters the system via one of two paths:
| Path | Source | Entry point |
|------|--------|-------------|
| **Simulation** | `fakeData.js` generates realistic pollution reports every 8 seconds | `setInterval` → `processTweetDetailed()` |
| **Citizen report** | User submits text/image/video via AddTweetModal | `POST /api/tweet` → `processTweetDetailed()` |

### 2. Multi-Modal Analysis (Gemini Vision)
If a user uploads an image or video, it is sent to **Gemini 3.5 Flash** (`imageAnalysis.js`):
- **Images:** Analyzed for garbage burning, industrial smoke, construction dust, illegal dumping, or smog.
- **Videos:** Client-side extracts up to 10 keyframes and sends them sequentially to Gemini for a holistic assessment of the incident.
Gemini returns a strict JSON classification including pollution type and confidence.

### 3. NLP Pipeline (`nlpPipeline.js`)
If text is provided (or generated by AI from the image), it runs through the NER engine:
- `classifyPollution()`: Keyword matching across 6 taxonomy categories (e.g. `garbage_burning`, `industrial_smoke`).
- `extractLocations()`: Matches against curated Guwahati localities.
- `calculateRelevancy()`: A 5-signal noise filter (Text Quality, Source Credibility, Cross-Reference, Temporal Coherence, Engagement) to ignore sarcastic/promotional posts.

### 4. Multi-Source Fusion (`sensorGrid.js`)
Real-world AQI data (from OpenAQ/CPCB) is fused with social media reports. 
- If a fire is reported near a sensor, the sensor's internal "pollution pressure" spikes, predicting an AQI rise before the physical sensor registers the change.

### 5. Hotspot Clustering (`hotspotEngine.js`)
Events are grouped into spatial clusters (500m grid). The system calculates the predominant pollution type and severity for each cluster, rendering pulsing radar overlays on the map.

### 6. 24h Prediction (`predictionEngine.js`)
A deterministic model forecasts AQI for the next 24 hours based on:
1. Current sensor AQI baseline
2. Diurnal (time-of-day) traffic trends
3. Local weather (temperature & wind dispersion via Open-Meteo)
4. Active hotspots (event pressure)

### 7. Municipal Intelligence (`municipalEngine.js`)
When the Command Center is opened, the backend sends the top active hotspots, worst predictions, and weather data to the **Gemini API**. Gemini returns 3-5 highly actionable, prioritized interventions (e.g., "Deploy Water Mist Cannons to Fancy Bazaar") formatted for municipal dispatch teams.

### 8. Broadcasting
Processed events are stored in `EventStore` and broadcast via **Server-Sent Events (SSE)** to all connected clients (`/api/events/stream`).

## Translation & Multilingual Support

The system handles non-English input natively:
- **Server-side (NLP):** Every incoming report passes through `translator.js` (Google Translate API). The translated English text is classified, ensuring keywords are detected across Hindi, Assamese, Bengali, etc.
- **Client-side (UI):** Original text is displayed in the feed. A "Translate" button allows toggling. No additional API call is needed on the frontend.
